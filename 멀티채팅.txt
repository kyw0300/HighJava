- 서버  --> 서버에 접속된 클라이언트 목록은 Map을 이용하여 관리한다.
          --> Map의 key값은 '대화명', value값은 '클라이언트와 연결된 소켓객체' 로 한다.
  1. accept()메서드를 이용하여 클라이언트의 접속을 기다린다
  2. 클라이언트가 접속이 완료되면 클라이언트가 보내온 '대화명'을 받는다.
  3. 받은 '대화명'이 중복되는지 여부를 검사해서 검사 결과를 전송한다.
     (예 : 중복될경우 => '대화명중복', 중복되지않을경우 => 'OK' )
  4. 대화명이 중복될 경우에는 2번부터 다시 진행한다.
  5. 대화명이 중복되지 않으면 '대화명'과 소켓객체를 Map에 추가한다.
  6. 해당 소켓에서 전송되어온 메시지를 받아서 Map에 저장된 전체 클라이언트들에게 전송한다.(이 작업을 끝까지 계속한다...)



- 클라이언트
  1. 소켓객체를 생성해서 서버에 접속 요청을 한다.
  2. 접속이 완료되면 '대화명'을 입력 받아서 서버로 전송한다.
  3. 서버에서 보내온 대화명의 중복 여부 결과를 받는다.
  4. 결과가 '대화명중복'이면 2번부터 다시 진행한다.
  5. 결과가 'OK'이면 
	메시지를 입력받아 전송하는 작업과
        다른 사람이 보낸 메시지를 받아서 화면에 출력하는 작업을 수행한다.
